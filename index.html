Input Data

Memahami data secara menyeluruh sebelum menyentuh model. Data adalah fondasi segalanya — garbage in, garbage out.

Apa itu Data?

Definisi

Data adalah representasi fakta, observasi, atau pengukuran dari dunia nyata yang direkam dalam format tertentu. Data mentah belum memiliki makna sampai dianalisis — ia adalah bahan baku Machine Learning.

Format & Struktur Data

📄 Tabular / Structured

Data dalam baris dan kolom. Format: CSV, Excel (XLSX), TSV, SQL database, Parquet, Feather, HDF5, ORC.

  • Paling umum dalam ML klasik
  • Mudah dibaca pandas/SQL
  • Tiap baris = satu observasi

🌐 Semi-Structured

Punya skema tapi fleksibel. Format: JSON, XML, YAML, TOML, HTML, API responses.

  • Bisa nested / hierarkis
  • Butuh parsing sebelum dipakai
  • Umum di web scraping & API

🖼️ Unstructured

Tidak ada skema baku. Format: JPEG/PNG (gambar), MP3/WAV (audio), MP4 (video), PDF, TXT.

  • Butuh ekstraksi fitur (CNN, BERT)
  • Volume terbesar di dunia nyata
  • Deep Learning dominan di sini

📡 Time Series / Stream

Data berurutan berdasarkan waktu. Format: sensor data, log files, stock prices, IoT data.

  • Urutan dan timestamp penting
  • Sering memerlukan windowing
  • Trend, seasonality, stationarity

🕸️ Graph / Network

Data relasional dengan node dan edge. Format: GraphML, GML, edgelist, adjacency matrix.

  • Social network, knowledge graph
  • GNN untuk pemrosesan
  • Hubungan antar entitas kunci

🗄️ Database Format

Data tersimpan dalam DBMS. Format: SQL (MySQL, PostgreSQL), NoSQL (MongoDB, Cassandra), Data Warehouse (BigQuery, Snowflake).

  • Akses via query language
  • ACID transactions
  • Bisa sangat besar (TB/PB)

Sifat Data: Kualitatif vs Kuantitatif

🔤 Kualitatif (Kategorik)

Menggambarkan kualitas atau kategori, tidak bisa dihitung langsung.


Nominal: Tanpa urutan. Contoh: warna, kota, jenis produk, nama merek.


Ordinal: Ada urutan tapi jarak tidak sama. Contoh: "rendah-sedang-tinggi", rating bintang 1-5, level pendidikan.

🔢 Kuantitatif (Numerik)

Menggambarkan jumlah atau besaran, bisa dihitung secara matematis.


Diskrit: Nilai bulat terhitung. Contoh: jumlah anak, jumlah transaksi, berapa kali login.


Kontinu: Nilai tak terhingga dalam range. Contoh: tinggi badan, suhu, harga, durasi.

Dimensi Kualitas Data

✅ Accuracy

Seberapa benar nilai data merepresentasikan kenyataan. Nilai usia -5 atau 999 tahun jelas tidak akurat.

📏 Completeness

Seberapa lengkap data — tidak ada missing values yang seharusnya ada.

🔄 Consistency

Data tidak kontradiksi antar sumber. "Jakarta" vs "JAKARTA" vs "Jkt" untuk kota yang sama.

⏰ Timeliness

Data cukup terkini untuk tujuan analisis. Data harga rumah 2010 tidak valid untuk prediksi 2025.

🎯 Relevance

Data yang dikumpulkan memang relevan dengan problem yang ingin dipecahkan.

🔁 Uniqueness

Tidak ada duplikasi data yang tidak diinginkan. Satu transaksi tercatat dua kali.

Data Preprocessing

Transformasi data mentah menjadi data bersih dan siap model. Ini adalah tahap yang paling memakan waktu — 60–80% waktu proyek ML ada di sini.

1. Handling Missing Values

Tiga Mekanisme Missing Data

MCAR (Missing Completely At Random): Hilang secara acak, tidak berhubungan dengan data lain. Aman dihapus.
MAR (Missing At Random): Hilang berkaitan dengan variabel lain yang terobservasi. Bisa diimputasi.
MNAR (Missing Not At Random): Hilang berkaitan dengan nilai itu sendiri. Butuh pendekatan khusus.

StrategiKapan PakaiKelebihanKekurangan
Hapus barisMissing < 5%, MCAR, dataset besarSimpel, tidak bias jika MCARKehilangan data, bias jika bukan MCAR

Mean/Median imputationNumerik, distribusi normal/skewedCepat, tidak hapus dataMengurangi variance, distorsi distribusi
Mode imputationKategorikCepatBisa bias jika distribusi tidak unimodal

5. Feature Engineering

💡 Mengapa Feature Engineering Penting?

Model terbaik dengan fitur biasa akan kalah dengan model biasa + fitur yang tepat. Feature engineering adalah seni mengubah raw data menjadi representasi yang bermakna bagi model.

📅 Datetime Features

  • Ekstrak: tahun, bulan, hari, jam, menit
  • Hari dalam seminggu (0-6), hari libur
  • Kuartal, semester, bulan ke-N
  • Selisih waktu (days_since_last_order)
  • Cyclical encoding (sin/cos untuk jam)

🔢 Fitur Agregasi

  • Mean, sum, count per group (groupby)
  • Rolling window statistics
  • Lag features (nilai t-1, t-7, t-30)
  • Expanding window (cum mean, cum sum)
  • Percentile rank dalam grup

🔗 Interaksi Fitur

  • Perkalian dua fitur (polynomial)
  • Rasio (harga_per_m2 = harga / luas)
  • Perbedaan (delta = nilai_saat_ini - baseline)
  • PolynomialFeatures dari sklearn

📝 Text Features

  • Panjang teks, jumlah kata, jumlah kalimat
  • TF-IDF, Bag of Words
  • Word embeddings (Word2Vec, GloVe)
  • Sentiment score, readability score
  • Named entity count

🗺️ Geospasial Features

  • Jarak ke landmark penting (Haversine)
  • Cluster geografis (K-Means)
  • Geohash encoding
  • Populasi dalam radius tertentu

📦 Domain-Specific

  • RFM (Recency, Frequency, Monetary)
  • BMI dari berat + tinggi
  • Debt-to-income ratio
  • Feature dari subject matter expert

8. Batch, Epoch, dan Iteration

📦 Batch

Subset dari training data yang diproses sekaligus dalam satu forward/backward pass. Contoh: dari 10.000 data, batch_size=32 → proses 32 data sekaligus.

🔁 Epoch

Satu kali model melihat seluruh training dataset. 1 epoch = (total data / batch_size) iterations. Training biasanya 10–1000 epoch.

⚡ Iteration

Satu kali update parameter (satu batch). Total iteration = epoch × (N / batch_size). Yang menggerakkan gradient descent.

🎯 Batch Gradient Descent

Gunakan SEMUA data untuk hitung gradient. Akurat tapi sangat lambat untuk data besar. Hampir tidak pernah dipakai.

⚡ Stochastic GD (SGD)

Gunakan 1 data point per update. Sangat noisy, sering konvergen ke suboptimal. Update sangat cepat.

🏆 Mini-Batch GD

Gunakan batch kecil (32, 64, 128, 256). Balance antara kecepatan dan stabilitas. Standar di deep learning.

📌 Pilih Batch Size

Batch kecil (16-64): Noisy gradient, regularisasi alami, konvergensi ke generalisasi lebih baik. Cocok saat dataset kecil atau GPU terbatas.
Batch besar (256-4096): Gradient lebih stabil, training lebih cepat per epoch, butuh learning rate tuning lebih hati-hati. Cocok untuk GPU kuat dan large-scale training.

Model Selection & Modelling

Memilih algoritma yang tepat adalah ilmu sekaligus seni. Tidak ada model terbaik secara universal — semua bergantung pada sifat data dan tujuan masalah.

Flowchart Pemilihan Model

START: Apa tujuan masalahmu?
├─ ADA LABEL (Supervised)
├─ Output: ANGKA KONTINU → Regresi
├─ Linear relationship → Linear/Ridge/Lasso
├─ Non-linear, tabular data → XGBoost / LightGBM
└─ Time series → ARIMA / Prophet / TFT
└─ Output: KATEGORI → Klasifikasi
├─ Butuh interpretasi → Logistic Reg / Decision Tree
├─ Tabular data umum → Random Forest / XGBoost
├─ Data teks → BERT / Naive Bayes
└─ Data gambar → CNN / ViT
└─ TIDAK ADA LABEL (Unsupervised)
├─ Grouping data → Clustering (K-Means, HDBSCAN)
├─ Kurangi dimensi → PCA / UMAP / t-SNE
├─ Temukan pola → Association Rules (FP-Growth)
└─ Generasi data baru → GAN / VAE / Diffusion

🎓 No Free Lunch Theorem

Tidak ada algoritma yang universally terbaik untuk semua masalah. Setiap model membuat asumsi tertentu tentang data (inductive bias). Kunci: pahami asumsi model dan sesuaikan dengan sifat data kamu. Selalu benchmark beberapa model dan pilih berdasarkan evidence.

Tahap Kelima

Model Evaluation

Mengukur performa model secara objektif. Metric yang salah bisa membuatmu percaya model bagus padahal buruk di dunia nyata.

Metric untuk Klasifikasi

Confusion Matrix — Fondasi Semua Metric Klasifikasi

TP (True Positive), FP (False Positive), TN (True Negative), FN (False Negative)

Accuracy
(TP+TN) / (TP+FP+TN+FN)
Proporsi prediksi benar. Menyesatkan untuk imbalanced dataset.
Precision
TP / (TP + FP)
Dari semua prediksi positif, berapa yang benar. Penting jika FP mahal.
Recall (Sensitivity)
TP / (TP + FN)
Dari semua positif aktual, berapa yang tertangkap. Penting jika FN mahal.
F1 Score
2 × (P×R) / (P+R)
Harmonic mean precision & recall. Untuk imbalanced classes.
ROC-AUC
Area under ROC curve
Kemampuan ranking: probability prediksi positif lebih tinggi dari negatif.
PR-AUC
Area under PR curve
Lebih informatif dari ROC-AUC untuk sangat imbalanced dataset.
Log Loss
-Σ y·log(p) / N
Mengukur kalibrasi probabilitas. Semakin kecil semakin baik.
Matthews CC (MCC)
(TP·TN-FP·FN) / √...
Metric paling balanced untuk imbalanced, range [-1, +1].
Cohen's Kappa
(P_o - P_e) / (1 - P_e)
Akurasi yang disesuaikan chance agreement. Untuk multi-class.
Kapan Pakai Metric Apa?

Fraud/Cancer Detection (FN mahal): Prioritaskan Recall
Spam Filter (FP mahal): Prioritaskan Precision
Imbalanced classes: F1-Score, PR-AUC, MCC
Butuh ranking/threshold agnostic: ROC-AUC
Multi-class: Macro/Micro/Weighted F1

Metric untuk Regresi

MAE
Σ|y - ŷ| / N
Mean Absolute Error. Robust terhadap outlier. Dalam unit asli.
MSE
Σ(y - ŷ)² / N
Mean Squared Error. Menghukum error besar lebih keras. Sensitif outlier.
RMSE
√MSE
Root MSE. Dalam unit asli seperti MAE tapi lebih sensitif outlier.
R² Score
1 - SS_res/SS_tot
Proportion of variance explained. Range (-∞, 1]. 1 = perfect.
MAPE
Σ|y-ŷ|/|y| / N × 100%
Percentage error. Mudah diinterpretasi bisnis tapi bermasalah jika y mendekati 0.
Huber Loss
MAE jika |e|>δ, MSE jika ≤δ
Hybrid MAE-MSE. Robust outlier tapi masih differentiable.

Bias-Variance Tradeoff & Diagnostik

🔴 Overfitting (High Variance)

Training error rendah, validation error tinggi. Model hafal training data tapi gagal generalisasi.

Solusi: Regularisasi (L1/L2), dropout, early stopping, lebih banyak data, data augmentation, kurangi kompleksitas model, cross-validation.

🟡 Underfitting (High Bias)

Training dan validation error sama-sama tinggi. Model terlalu sederhana untuk menangkap pola.

Solusi: Model lebih kompleks, lebih banyak fitur, feature engineering lebih baik, kurangi regularisasi, training lebih lama.

Tahap Keenam

Model Fitting & Optimization

Mengoptimalkan model agar mencapai performa terbaiknya. Ini adalah siklus iteratif antara tuning, evaluasi, dan penyesuaian.

Hyperparameter vs Parameter

⚙️ Parameter (Dipelajari dari Data)

Nilai yang dioptimalkan otomatis selama training via gradient descent atau optimization algorithm.

Contoh: bobot neural network (W, b), koefisien regresi (β), split threshold decision tree.

🎛️ Hyperparameter (Kita yang Set)

Konfigurasi yang ditentukan sebelum training dan tidak dipelajari dari data. Harus di-tuning manual atau otomatis.

Contoh: learning rate, n_estimators, max_depth, dropout rate, batch size, kernel type.

Metode Hyperparameter Tuning

🔲 Grid Search

Exhaustive search semua kombinasi dari grid yang didefinisikan. Pasti temukan terbaik dalam grid tapi sangat lambat secara eksponensial.

  • Cocok untuk ≤ 3 hyperparameter
  • Kombinasi kecil yang diketahui

🎲 Random Search

Sample secara acak dari distribusi hyperparameter. Lebih efisien dari Grid Search — dengan budget sama, explore lebih banyak.

  • Lebih baik dari Grid untuk banyak HP
  • n_iter mengontrol budget

🧠 Bayesian Optimization

Bangun surrogate model (Gaussian Process) dari hasil sebelumnya untuk predict HP mana yang paling menjanjikan. Sangat efisien.

  • Tools: Optuna, Hyperopt, Scikit-Optimize
  • Terbaik untuk fungsi mahal

⚡ Halving Search

Mulai dengan banyak kandidat + sedikit data → eliminasi yang buruk → survivors dapat lebih banyak data. Sangat cepat.

  • HalvingGridSearchCV di sklearn
  • Cocok untuk dataset besar

🏃 Hyperband

Iterasi successif halvings dengan berbagai budget. Generalisasi Successive Halving yang lebih principled.

  • Sangat efisien untuk DL
  • Dipakai di Keras Tuner, Ray Tune

🤖 AutoML

Otomasi seluruh pipeline: feature engineering + model selection + hyperparameter tuning sekaligus.

  • Tools: AutoSklearn, TPOT, H2O AutoML
  • Bagus untuk baseline cepat

Learning Rate & Optimizers (untuk Deep Learning)

📐 Apa itu Learning Rate?

Learning rate (α) menentukan seberapa besar langkah update parameter pada setiap iterasi gradient descent: θ ← θ - α × ∇L(θ). Terlalu besar: diverge/oscillate. Terlalu kecil: konvergensi sangat lambat atau stuck di local minima.

Learning Rate Scheduling

Step Decay

Turunkan LR dengan faktor tetap setiap N epoch. Sederhana dan efektif.

Cosine Annealing

LR mengikuti kurva cosine. Smooth, konvergensi baik. Populer untuk training besar.

Warmup + Decay

Mulai LR kecil → naik → turun. Standar untuk training Transformer (BERT, GPT).

Cyclic LR

Naik turun periodik antara batas. Bisa escape local minima. CLR dari Smith et al.

ReduceOnPlateau

Turunkan LR otomatis jika val loss tidak improve selama N epoch. Adaptif.

1-Cycle Policy

LR naik lalu turun dalam satu siklus. Fast convergence, digunakan di fastai.

Tahap Ketujuh

Model Deployment

Model yang tidak di-deploy tidak memberi nilai. Deployment adalah proses membawa model dari notebook ke production yang dipakai pengguna nyata.

📌 Apa itu Model Deployment?

Proses mengintegrasikan model ML yang sudah dilatih ke dalam sistem production sehingga bisa menerima input real-world, membuat prediksi, dan mengembalikan hasil ke pengguna atau sistem lain — secara otomatis, skalabel, dan andal.

Pola Deployment

⚡ Real-Time API (Online Serving)

Model dibungkus REST API. Request masuk → prediksi → response dalam milidetik.

  • Latency: < 100ms biasanya
  • Tools: FastAPI, Flask, TorchServe
  • Use case: fraud detection, rekomendasi live, chatbot

📦 Batch Prediction

Prediksi dilakukan secara bulk untuk banyak data sekaligus, biasanya terjadwal (harian/mingguan).

  • Latency: menit sampai jam
  • Tools: Spark, Airflow, AWS Batch
  • Use case: scoring jutaan pelanggan tiap malam

📱 Edge Deployment

Model di-deploy langsung di perangkat (HP, IoT, embedded) tanpa koneksi server.

  • Butuh model compression (quantization)
  • Tools: TensorFlow Lite, ONNX, CoreML
  • Use case: face unlock HP, deteksi wake word Siri

🌊 Streaming Prediction

Prediksi real-time pada data stream yang terus mengalir secara kontinu.

  • Tools: Apache Kafka + Flink, Spark Streaming
  • Use case: anomaly detection pada log, fraud on transaction stream

# main.py — Simple ML Model API from fastapi import FastAPI from pydantic import BaseModel import joblib import numpy as np app = FastAPI() model = joblib.load("model.pkl") # load model saat startup scaler = joblib.load("scaler.pkl") # load scaler yang sama saat training! class PredictRequest(BaseModel): feature1: float feature2: float feature3: str @app.post("/predict") def predict(req: PredictRequest): features = np.array([[req.feature1, req.feature2]]) scaled = scaler.transform(features) # transform pakai scaler training prediction = model.predict(scaled) probability = model.predict_proba(scaled) return { "prediction": int(prediction[0]), "probability": float(probability[0][1]), "model_version": "v1.2.0" }

Model Monitoring di Production

📊 Data Drift

Distribusi input berubah dari training data. Model masih berjalan tapi hasilnya tidak valid lagi. Deteksi via KS test, PSI, Wasserstein distance.

🎯 Concept Drift

Hubungan antara X dan y berubah (dunia berubah). Misalnya pola fraud berubah setelah regulasi baru. Perlu retrain atau continuous learning.

⚡ Performance Monitoring

Pantau latency, throughput, error rate, memory usage, CPU/GPU utilization secara real-time.

📈 Business Metrics

Pantau metrik bisnis yang didorong model: CTR, conversion rate, churn prevention rate, recall on fraud.

🔄 Model Versioning

Track semua versi model, eksperimen, dan artifact. Tools: MLflow, DVC, Weights & Biases, Neptune.

🔁 Retraining Strategy

Kapan retrain? Scheduled (setiap minggu), trigger-based (saat drift terdeteksi), atau continuous learning stream.
